## ADDED Requirements

### Requirement: 阅读者是全知角色

阅读者 SHALL 是面试模块中唯一同时持有被测项目源码、用户简历履历与岗位 JD 的角色。系统 SHALL 接收用户提供的简历履历文件与岗位 JD 文件路径,读取失败时以非零退出码报错。

阅读者的全知 MUST NOT 通过任何接口原样外泄给裁判或面试官。

#### Scenario: 三份输入齐备时正常启动

- **WHEN** 用户提供合法的被测项目仓库路径、简历履历文件与岗位 JD 文件
- **THEN** 阅读者读入三者并可执行离线生成与在线问答

#### Scenario: 简历或 JD 文件不存在时报错退出

- **WHEN** 传入的简历履历文件或岗位 JD 文件在文件系统中不存在
- **THEN** 系统以非零退出码终止,并打印指明哪份文件缺失的清晰错误信息

### Requirement: 阅读者注册检索与读取工具

阅读者 SHALL 注册 `glob(模式)`、`grep(模式[, 路径])` 与 `read_file(路径)` 三个工具,用于在被测项目中检索与读取源码。

这三个工具的路径与模式参数均为模型输出,属不可信输入。实现 MUST 把解析后的绝对路径限制在已声明的仓库根之内,拒绝 `..` 逃逸、符号链接与仓外绝对路径。

三个工具 MUST 各自设置输出上限,且超限时的响应方式 MUST 按工具性质区分:

- `read_file` 超出文件体积上限时 MUST 抛出可读错误并提示改用范围读取,MUST NOT 返回截断后的内容。理由:错误结果只占约一百字节,而截断内容会占满上限本身,后者对下游上下文的消耗高出两个数量级。为使该提示可执行,`read_file` MUST 支持按行偏移与行数的范围读取。
- `grep` 与 `glob` 超出结果条数上限时 MUST 返回上限内的结果并告知已截断——搜索本就预期多命中,抛错等于什么都没返回。

截断信息 MUST 只在实际发生截断时给出:命中数恰好等于上限而未丢弃任何结果时 MUST NOT 报告截断,否则会诱导下游去取一页空结果。告知截断时 MUST 附带下一步建议(缩小模式,或以偏移量继续取下一页)。

`grep` 与 `glob` MUST 提供解除条数上限的显式入参。理由:判定"某项实现不存在"要求穷尽检索,而默认上限之下无法排除被丢弃的结果里仍有命中,该判定不成立。

#### Scenario: 检索命中时返回结果

- **WHEN** 阅读者以某个模式调用 `grep`,被测项目中存在命中
- **THEN** 工具返回命中的文件与行,数量不超过上限

#### Scenario: 结果超过上限时截断并给出下一步建议

- **WHEN** 某次 `grep` 的命中数超过返回上限
- **THEN** 工具返回上限内的结果,说明结果已被截断,并给出缩小模式或以偏移量翻页的建议

#### Scenario: 命中数恰好等于上限时不报告截断

- **WHEN** 某次 `grep` 的命中数恰好等于返回上限,没有任何结果被丢弃
- **THEN** 工具返回全部结果,不报告截断

#### Scenario: 读取超体积上限的文件时抛错而非截断

- **WHEN** 阅读者请求读取的文件体积超过上限
- **THEN** 工具抛出可读错误并提示改用范围读取,不返回该文件被截断后的内容

#### Scenario: 解除上限以穷尽检索

- **WHEN** 阅读者需要判定某项实现是否存在,以解除上限的入参调用 `grep`
- **THEN** 工具返回全部命中而不施加条数上限,使"不存在"的判定得以成立

#### Scenario: 拒绝仓外路径

- **WHEN** 工具收到一个解析后落在已声明仓库根之外的路径(含 `..` 逃逸与仓外绝对路径)
- **THEN** 工具拒绝该次调用并返回可读的错误信息,不读取该文件

#### Scenario: 读取不存在的文件不中断

- **WHEN** 阅读者请求读取一个在被测项目中不存在的路径
- **THEN** 工具返回可读的错误信息给阅读者,阅读者据此调整,整体流程不崩溃

### Requirement: 在线问答接口只返回自然语言结论

阅读者 SHALL 提供 `ask_reader(question)` 接口,供裁判在面试过程中就地图未覆盖的问题提问。

该接口的返回值 MUST 是自然语言结论,MUST NOT 包含源码片段、代码块或文件路径列表。该约束 MUST 由返回值的类型与接口形态保证——接口 MUST NOT 提供任何返回源码或结构化代码引用的模式,MUST NOT 依赖调用方或模型"记得不要贴代码"。

#### Scenario: 回答地图未覆盖的问题

- **WHEN** 裁判就一个 `GRILL.md` 中未记载的实现细节调用 `ask_reader`
- **THEN** 阅读者检索源码后返回一段自然语言结论,其中不含源码片段或代码块

#### Scenario: 确认某项实现不存在

- **WHEN** 裁判询问某项功能是否已实现,而阅读者检索后在项目中找不到对应实现
- **THEN** 阅读者返回明确指出该实现不存在的自然语言结论

#### Scenario: 接口不提供返回源码的模式

- **WHEN** 检视 `ask_reader` 的接口定义
- **THEN** 其返回值类型只允许自然语言文本,不存在可请求源码原文或文件路径列表的参数或返回分支

### Requirement: 阅读者的 Prompt 只承载理解力

阅读者的 Prompt SHALL 只描述如何理解一个项目、如何取舍写进地图的内容。产出结构 MUST 由 schema 约束而非 Prompt 描述;信息边界 MUST 由工具注册与返回值类型决定而非 Prompt 约定。

#### Scenario: 结构约束不依赖 Prompt 措辞

- **WHEN** 阅读者 Prompt 中关于输出格式的描述被移除
- **THEN** 产出结构仍由 schema 强制保证,生成结果仍通过校验
