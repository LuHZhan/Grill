# Grill 风格规范 —— 「对谈实录型」(源自 2a)

> 参考实现：`interview-style-comparison.dc.html` 中 `#2a`。后续所有界面以此为准。

## 气质定位
陪练教练：专业、克制、站在用户这边；有张力但不压迫。
不是聊天软件 —— 是一场"被记录的对谈"。产品概念直接做成界面语言。

## 色板
- 纸面底：`#fbfaf8`（暖纸白）；分区底 `#f6f4f0`；边框发丝线 `#edebe6` / `#e2dfd8`
- 墨色：`#17161a`（标题/强文本）；正文 `#54515c`；弱文本 `#8a8690` / `#b4b1ab`
- 强调（信任/操作）：靛紫 `oklch(0.52 0.16 285)`，深态 `oklch(0.45 0.14 285)`
- 张力（接缝/追问/PARTIAL）：琥珀 `oklch(0.68-0.72 0.14 55-60)`，高亮底 `oklch(0.93 0.06 60)`
- 稳（SOLID）：绿 `oklch(0.68 0.13 155)`；崩（COLLAPSED）：红 `oklch(0.72 0.19 25)` 仅小面积
- 深色元素（语音胶囊等）：`#17161a` 底 + `#44414c/#5c5966/#8a8696` 灰阶

## 字体
- 面试官提问 / 判定大字：**Noto Serif SC** 700，24–28px，line-height 1.55（对谈实录感）
- 正文 / UI：**Manrope**（西文数字）+ **Noto Sans SC**，`'Manrope','Noto Sans SC',sans-serif`
- 代码 / 轮次 / 标签 / 品牌字：**JetBrains Mono**，小号大字距（letter-spacing 1.5–3px）
- 小标签样式：10.5–11px，700，letter-spacing 1.5px，弱灰色

## 签名组件（产品概念 → 界面语言）
1. **追问链纵轴**：左侧 88px 脊柱；已过轮 = 实心圆（绿/琥珀按判定着色），当前轮 = 放大黑圆 + 靛紫光环，未来轮 = 虚线空圆；底部"挖掘层 -N"。
2. **接缝高亮**：引用用户原话（左 3px 灰竖线），脆弱短语用琥珀 `<mark>` + 2px 下边线；下一行 `└─ 裁判没找到落点` 注解。
3. **两分差距仪表**：一条刻度、两根指针（抗压琥珀 / 表达靛紫），中间斜纹带 + `GAP N.N`（JetBrains Mono）。
4. **语音胶囊**：深色 `#17161a` 圆角 40px 悬浮胶囊：红点脉冲 + 计时 + 波形（3px 竖条）+ 实时转写斜体 + ⌨/↑ 圆钮。
5. **裁判档案**：右侧 320px，底 `#f6f4f0`；判定词用衬线大字，"最脆的接缝 / 下钻记录 / 教练一句话"分节，mono 文件清单 ✓。
6. **巨型轮次水印**：`Q3` JetBrains Mono 150px，opacity 0.04。

## 微文案口吻
教练式、站在用户这边、短句：
"说不上来就直说'没做'——诚实比包装扛揍。" / "这里没有观众，练崩了不算数。" / "删掉'完整的'三个字，抗压分马上回来。"

## 禁忌
- 不用 IM 聊天气泡铺满的布局；不用彩色渐变大底；无 emoji（⌨/↑ 等功能符号除外）
- 红色只在"崩/录音中"小面积出现；圆角体系 12–24px；阴影只做柔和分层
